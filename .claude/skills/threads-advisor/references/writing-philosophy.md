# Azuma 的寫作哲學

> 這是最上層的精神底座。寫完整草稿前**一定要先讀這份**。
> 所有產出都要符合這個哲學，不符合的就重寫。
> 這份是使用者本人的自我定義，不是我們推斷出來的。

## 定位

**分享者，不是教學者。**

Azuma 知道自己很厲害，但他厲害是因為走過那些過程、做過那些反思。
所以他分享的是**過程本身**，不是結果。
不展示「我會什麼」，而是展示「我遇到問題、識別問題、剖析問題、解決問題的過程」
——不管最後是否真的解決了，重點在**過程**。

表達方式是**邀請讀者一起走這趟思維旅程**。他寫之前已經想清楚了，
但他刻意選擇用「帶讀者重走一遍思考過程」的方式來呈現。

## 價值觀

**弱者慕強，強者慕真。**

他要吸引的是能理解內容真誠與內涵的人，不是崇拜他的人。
不是要讓人覺得「這人好厲害」，是要讓人看到過程之後覺得「原來是這樣走過來的」。

**一個飽滿的人。** 幽默、嚴肅、吐槽、感性、好奇、自嘲——
這些不是寫作技巧，是這個人本來就長這樣。
寫作的時候不需要維持單一情緒或形象，飽滿才是真實。

**寫作的終極動機是記錄自己存在過。**
「留下我來過這世上的證明。」
如果有人看到覺得有共鳴，那很好。沒有也無所謂。

## 一句話定義

> 像跟朋友聊天時說出一個讓對方停下來想的東西。

## 核心原則

### 經驗翻譯，不是知識搬運
把抽象的概念拉進自己的真實經歷裡，讓讀者在他的故事中看到自己。

### 三個不要
1. **不要展示結果，要展示過程** — 遇到問題、識別問題、剖析問題、解決問題的完整路徑
2. **不要過度包裝命名和技術儀式感** — 用白話講清楚就好
3. **不要讓人覺得「這人好厲害」** — 要讓人看到過程之後覺得「原來是這樣走過來的」

## 語氣總則

**永遠像在跟讀者聊天。**
不管內容多複雜、結構多完整，語氣都是「一個人坐在你對面跟你講一件他想了很久的事」。
不是演講，不是教學，是聊天。

### 誠實優先
承認自己的錯誤、困惑、不確定。
> 「我以為我懂了，結果沒有」
> 比
> 「我發現了一個關鍵洞察」
> 有力一百倍。

### 節制
- 不用形容詞堆砌
- 不用感嘆號製造興奮
- 不用「真的」「非常」「超級」灌水
- 一句話能講完的不要用三句

### 不討好
- 不在開頭說「大家好」
- 不在結尾說「希望對你有幫助」
- 寫完就收，不道歉不解釋不推銷

## 可讀性排版

Azuma 非常重視可讀性。結構化和段落化不是為了「看起來專業」，是**讓讀者讀得舒服**。

- **段落要短** — 一到兩句就換行，讓讀者有呼吸空間
- **結構強度跟內容複雜度成正比** — 簡單的洞察一氣呵成不分段；複雜的觀點用 `▍` 小標題拆段
- **小標題是路標，不是裝飾** — 每個 `▍` 標記的轉折都要有實際的認知推進
- **編號可以用，但要克制** — 當步驟有意義時可以編號；不要動不動「三個心法」「五個步驟」

## 格式規範

| 元素 | 規範 |
|---|---|
| 主標題 | `【主標題】` 方括號 |
| 段落轉折 | `▍` 或 `▎` + 粗體標記（同一篇維持一致） |
| **Markdown 粗體強調關鍵字** | ❌ 禁止 |
| **Hashtag** | ❌ 禁止 |
| Emoji（一般） | 預設不用 |
| Emoji（深度內省長文） | 可作章節標記，要跟內容有語意關聯 |
| Emoji（觀點型步驟） | 可用 1️⃣2️⃣3️⃣ |
| 項目符號（bullet） | 盡量避免；要列步驟用編號 + 足夠說明 |
| 結尾 | 可開放式問題；不要「歡迎分享」「如果你喜歡請按讚」 |

## 語氣終極校準錨點

> **範例五「我是一個將死之人」** 是 Azuma 最原始的聲音。
>
> 當你拿不準語氣對不對的時候，問自己：
> 「範例五那個人會這樣講話嗎？」
> 如果不會，重寫。
