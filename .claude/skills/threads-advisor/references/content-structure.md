# Azuma 的內容結構（Narrative Structure）

> 來源：使用者本人提供的內容結構指南，是經過累積整理的權威版本。
> 用途：`threads-advisor` skill 在寫完整草稿（Path A Step 5）時必讀，確保產出貼合使用者的敘事習慣。

## 核心：認知翻轉鏈

Azuma 的內容核心結構是一條**認知翻轉鏈**。每一篇都在帶讀者走一段：

> 我原本以為 → 被現實打臉 → 我現在的理解

這是預設結構，除非題材明確屬於其他型（分析評測、深度內省、自由書寫），否則都用這個。

---

## 完整結構（長文，500-1000 字）

```
【標題：用一句話勾起好奇心】

[錨點] — 一個具體的個人場景或經歷，讓讀者進入你的世界
         通常是過去的自己，做了某件事或相信了某件事

▍[第一次翻轉] — 現實打臉的具體場景
                不是抽象地說「我發現我錯了」
                而是描述那個讓你意識到不對的具體瞬間

[錯誤歸因] — 你當時怎麼解讀那次打臉
             通常是錯的，但你那時候不知道
             這段讓讀者產生共鳴：「我也會這樣想」

▍[轉折] — 某個事件、方法、或對話改變了你的理解
          具體描述是什麼改變了你

[新理解] — 你現在怎麼看這件事
           用最白話的方式講清楚
           如果涉及專業概念，用日常例子解釋

▍[再翻轉（可選）] — 新理解也不完整，又被現實修正
                    這層讓內容有深度，但不是每篇都需要

[落地] — 這個理解怎麼改變了你的行動或思考方式
         不要講大道理，講具體的變化

[開放結尾] — 留一個問題或缺口，讓讀者帶回自己的生活
```

---

## 短結構（社群短內容，300-500 字）

短內容只取完整結構中的一段翻轉。

```
【標題】

[場景] — 一個具體的情境，兩三句就好

[我以為] — 你原本的理解或假設，一兩句

[但其實] — 翻轉。新的理解。

[為什麼] — 用一個例子或類比解釋為什麼是這樣，而不是那樣

[所以呢] — 這個理解對你（或讀者）意味著什麼。不用太滿，點到為止。
```

## 系列文章結構（讀後感系列）

當內容是基於某篇文章或 podcast 的系列讀後感時：

**第零篇：開場**
- 介紹原始素材是什麼
- 說清楚這不是摘要，是你從裡面長出來的東西
- 預告會有什麼讓人想看下去的東西

**第 N 篇：各段讀後延伸**
- 先用一兩句講原作者這段在說什麼（不要超過三句）
- 然後切入你的延伸：你同意什麼、不同意什麼、補上了什麼
- 用你自己的經驗或場景佐證
- 每篇只聚焦一個核心洞察，不要貪多

## 分析評測結構（技術分析、工具比較）

```
【標題】

[脈絡] — 為什麼寫這篇。誰分享了什麼、你看到什麼、為什麼覺得值得分析。
         兩三句交代完，不囉嗦。

▋ [分析對象 A]
一句話人設定位（「天賦型選手」「黑馬型選手」）

數據面 — 用固定框架拆解（能力面、速度面、費用面、適合場景等）
         數據→白話翻譯→一句話結論，三層結構
         術語第一次出現加括號解釋

⇨ 我的心得 — 個人體感。用了什麼感覺怎樣、為什麼選或不選。
             吐槽直接講，不裝客觀。
             不熟的就說不熟：「這塊我使用經驗不深」

▋ [分析對象 B]
（同上框架）

▋ 結論
分享自己的選擇跟理由。不強迫讀者接受，「僅供參考」就好。
```

**關鍵**：「客觀分析」跟「主觀體感」要分開放。讀者可以只看分析段拿數據，也可以連心得段一起看拿你的判斷。混在一起講會讓兩邊都變弱。

## 深度內省結構（引用外部素材 + 個人領悟）

當內容是從別人的觀點出發，但核心是你自己的內在反思：

```
〈標題：點出核心矛盾或問題〉

[共鳴入口] — 引用外部素材（podcast、文章、別人的話）
             馬上接「我也有類似的感受」，把別人的話變成自己的入口
             引用是因為「他替我說出了什麼」，不是為了展示博學

【一、拆解矛盾】
把表面的現象拆開，看底層的衝突是什麼
用「我們」拉讀者進來，不只是講「我」

【二、深入矛盾】
矛盾的根源是什麼？為什麼會這樣？
可以連結到更普遍的人性困境

【三、轉向】
引入一個新的視角或錨點（可以是哲學、可以是自己的體悟）
關鍵場景：描述自己內在的真實對話或轉變時刻
這段要最真實，不能抽象

【四、連結當下（可選）】
把個人領悟擴展到更大的脈絡（如 AI 焦慮、時代困境）
從「我」擴展到「我們」

【五、和解 / 答案】
給出自己的答案，但不強迫
收尾可以比其他結構更明確，因為前面鋪了足夠的深度
開放式邀請讓讀者帶回自己的生活
```

這個結構的情感濃度最高。可以用 emoji 做章節標記，可以用稍正式的語氣，可以有文學性的句子。但底色還是聊天——「不知道大家是不是跟我一樣」。

## 自由書寫（隨筆型）

沒有模板。沒有結構。這不是一種「結構」，是一種狀態。

當內容純粹是個人記錄、生活感觸、不為讀者而寫的東西時，不要硬套任何框架。想到哪講到哪，岔題了就岔題，罵完了自己拉回來。

**什麼時候用**：當素材的核心是「我經歷了一件事，我想講出來」，而不是「我有一個觀點想傳達」的時候。前者不需要結構，後者才需要。

---

## 標題寫法

標題的功能是**製造好奇心缺口**——讓讀者想知道答案。

**好的標題**：
- 【我花了好幾年才明白，學習的真相是什麼？】
- 【Naval 說 AI 是心智的摩托車，我聽完之後想了很久】
- 【心力不是意志力，是摩擦力】

**不好的標題**：
- 【學習方法分享：從陳述性知識到程序性知識】
- 【Naval Ravikant AI 觀點精華整理】
- 【為什麼你應該重新思考心力的定義】

差異：好的標題讓你想點進去。不好的標題告訴你裡面有什麼，然後讀者覺得已經知道了，不用點。

---

## 段落間的過渡

**不要用過渡句。直接跳。**

❌ 不好：
> 「說完了學習的問題，接下來讓我們看看另一個面向。」

✅ 好：
直接開始下一段，用 `▍` 標記轉折就好。讀者會跟上來的。

---

## 引用他人觀點的方式

用自己的話轉述，不要直接翻譯原文。先講觀點，再講你的反應。

- 同意 → 說為什麼（用你的經驗）
- 不同意 → 說哪裡不同意（用你的邏輯）
- 補充 → 說你補了什麼、為什麼原本的不夠

**範例**：
> Naval 說學習的手段已經夠多了，稀缺的是學習的慾望。
>
> 我同意前半句，但覺得後半句不完整。
> 不只是慾望的問題。你有慾望，但如果學習方法不對，
> 你會像我以前學日文一樣——很認真，但認真錯了方向。

**不要這樣寫**：
> Naval Ravikant 在他的 podcast 中提到了一個重要觀點：
> 「The means of learning are abundant, what's scarce is the desire to learn.」
> 這句話深刻地揭示了當代學習的核心矛盾。

## Skill 使用備註

當使用者要求寫完整草稿時：
1. **先判斷結構**：這篇素材屬於哪一類（認知翻轉鏈 / 分析評測 / 深度內省 / 自由書寫 / 讀後感系列）
2. **鎖定對應結構**：不要混用
3. **結構名字不要寫進貼文**：讀者看到的是內容，不是框架名
4. **搭配 `voice-patterns.md`**：結構是骨架，語氣是血肉，兩個一起讀
