## V1：趨勢收集（已完成）

**流程**：Threads API `keyword_search` → 跨關鍵字去重 → `claude -p` AI 分析 → Jinja2 Markdown 日報

**分析維度**：
- 分類：新工具 / 產業動態 / 教學 / 觀點
- 評分：1–5 星趨勢分數

每天跑一次，輸出一份可閱讀的日報，讓自己快速掌握 AI/科技圈的討論動態。

---

## V2：成效追蹤（程式碼完成，待驗證）

**資料收集**：
- 抓所有自己的貼文 + 每篇 insights 數據（views, likes, replies, reposts, quotes）→ SQLite
- 抓帳號層級 insights（followers, total_views）

**分析輸出**：
- 7 天趨勢：粉絲成長、觀看變化
- Top 5 貼文排行（按觀看數）
- 合併趨勢日報 + 成效分析 → 戰情日報（dashboard）

**待驗證項目**：
- insights API 實際回傳格式
- rate limit 限制
- backfill 歷史資料的可行性

## V2.5：共鳴度分析（構想）

核心理念：「分享自己，不是推銷自己。讓讀者認識你是誰。」

在現有的趨勢分數基礎上，擴充 analyzer 的評分維度，加入「共鳴度分析」：

| 維度 | 說明 |
|------|------|
| 鉤子 | 有沒有讓人停下來的開頭 |
| 真實經驗／故事 | 不是空談，有沒有自己的實作或經歷 |
| Takeaway | 讀者看完能帶走什麼 |
| 聚焦度 | 1 篇文 = 1 個切入點、1 種人、1 個問題 |

靈感來源是「鉤子 → 故事 → 成交」的文案框架，但出發點是分享而非銷售。評分結果用於篩選值得參考的外部貼文，以及回顧自己過去哪些貼文真正引起共鳴。

## V3：議題分析 + 內容資產追蹤 + 分享靈感（構想）

### 議題分析

利用 `keyword_search` 已在收集的外部貼文數據：

- **熱門議題偵測**：哪些關鍵字／主題這週討論量爆增
- **議題生命週期**：從出現到退燒的時間軸
- **意見光譜**：同一議題大家的觀點分布（樂觀 / 悲觀 / 中立）

### 內容資產追蹤

- 追蹤每篇貼文的長期影響力，不只看發文當天的數據
- 辨識「常青內容」：發完後持續有人看的貼文
- 分析哪種主題／結構的貼文生命週期最長

### 分享靈感（交叉分析：外部趨勢 x 自己歷史成效）

- **議題缺口**：外面在討論但自己還沒寫的主題
- **差異化觀點**：自己的觀點跟主流不同的地方，這往往最有共鳴
- **時機建議**：某議題正在升溫，現在切入分享正好

## V4：全自動循環（願景）

```
收集趨勢 → AI 分析 → 建議發文 → AI 草稿 → 人工確認 → 自動發布 → 追蹤成效 → 調整策略 → 循環
```

- AI 根據趨勢分析和歷史成效，主動建議值得分享的主題
- 產出草稿後，由人工確認內容符合「分享自己」的出發點
- 發布後自動追蹤成效，數據回饋到下一輪的建議策略

人工確認這一關永遠存在。自動化的是資料流和草稿，不是判斷力。

---

## 設計前提

- **核心身份**：實際在用 AI 工具做事的人，分享過程和思考，不是在賣課或賣概念
- **內容策略**：不推銷，分享自己，讓讀者認識你是誰
- **Side project 原則**：先自己用好再說，不為了功能而功能
- **漸進式擴充**：每個 phase 都是上一個的自然延伸，不跳步、不過度設計
